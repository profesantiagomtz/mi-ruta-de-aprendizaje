01

¿Qué significa implementar herencia?

Implementar herencia significa pasar de la idea teórica a la construcción de código: crear una clase base, crear una clase derivada, instanciar objetos y comprobar que los métodos heredados funcionan correctamente.

💡 Explicación clara

En el inciso A identificamos la herencia. En este inciso B vamos a programarla. La pregunta ya no es “¿qué es?”, sino: ¿cómo la escribo en C++ y cómo la uso con objetos?

Objetivo práctico

Al terminar este material podrás construir aplicaciones pequeñas donde una clase hija use atributos y métodos de una clase padre, tanto en herencia simple como en herencia múltiple.

ElementoFunción en el programaEjemplo
Clase baseGuarda lo comúnPersona
Clase derivadaAgrega lo específicoAlumno
Objeto heredadoUsa métodos propios y heredadosAlumno a;
01.1

Antes de heredar: estructura base de un programa C++

Antes de programar la herencia, el alumno debe reconocer la estructura mínima de un programa en C++. Así no solo copia una clase, sino que sabe armar un archivo completo desde cero.

🧱 Paso cero

Todo ejemplo completo de esta clase se escribirá desde #include <iostream>, después using namespace std;, luego las clases y finalmente la función main(), que es donde el programa empieza a ejecutarse.

C++ · Plantilla mínima para iniciar desde cero
#include <iostream>
using namespace std;

int main() {
    cout << "Mi primer programa desde cero" << endl;

    return 0;
}
Estructura explicada línea por línea
#include <iostream>
Permite usar entrada y salida de datos, por ejemplo cout para mostrar mensajes en pantalla y cin para capturar datos.
using namespace std;
Evita escribir std:: antes de instrucciones como cout, cin, endl y string.
int main() {
Es la función principal. Todo programa en C++ inicia su ejecución aquí.
cout << "..." << endl;
Muestra texto en pantalla. endl baja a la siguiente línea.
return 0;
Indica que el programa terminó correctamente.
Regla para esta clase

Cada práctica debe entregarse como programa completo: librería, espacio de nombres, clases, función main(), creación de objetos y ejecución de métodos.

02

Herencia simple: Persona → Alumno

La herencia simple se implementa cuando una clase hija hereda de una sola clase padre. En C++ se escribe usando dos puntos y el tipo de herencia.

C++ · Herencia simple completa
#include <iostream>
using namespace std;

class Persona {
public:
    string nombre;

    void saludar() {
        cout << "Hola, soy " << nombre << endl;
    }
};

class Alumno : public Persona {
public:
    string matricula;

    void mostrarMatricula() {
        cout << "Matrícula: " << matricula << endl;
    }
};

int main() {
    Alumno a;
    a.nombre = "Luis";
    a.matricula = "A001";
    a.saludar();
    a.mostrarMatricula();
    return 0;
}
Código explicado línea por línea
#include <iostream>
Carga la librería necesaria para usar cout, cin y endl.
using namespace std;
Permite escribir cout y string directamente sin anteponer std::.
class Persona {
Crea la clase base. Aquí se coloca la información común que puede compartir cualquier persona.
string nombre;
Declara un atributo público llamado nombre. Como está en la clase padre, la clase hija podrá usarlo.
void saludar()
Crea un método de la clase padre. Este método será heredado por Alumno.
class Alumno : public Persona {
Crea la clase hija Alumno. La parte : public Persona indica que Alumno hereda públicamente de Persona.
string matricula;
Agrega un atributo propio de la clase hija. Una persona no siempre tiene matrícula, pero un alumno sí.
Alumno a;
Crea un objeto de tipo Alumno. Ese objeto tendrá lo heredado de Persona y lo propio de Alumno.
a.saludar();
Ejecuta un método heredado. Aunque saludar() fue escrito en Persona, el objeto Alumno puede usarlo.
03

Instanciar objetos heredados

Un objeto de una clase derivada puede usar miembros propios y miembros heredados. Esto permite crear programas más limpios porque no repetimos lo que ya existe en la clase base.

🍎 Dicho fácil

Si Alumno hereda de Persona, entonces al crear Alumno a;, ese objeto ya trae lo de Persona. Por eso podemos usar a.nombre y a.saludar().

Regla práctica

Cuando pruebes herencia, verifica dos cosas: que el objeto pueda usar lo heredado y que también pueda usar lo propio.

04

Constructores en herencia simple

Cuando una clase hija hereda de una clase padre que tiene constructor, la clase hija debe llamar al constructor de la clase padre para inicializar correctamente la parte heredada del objeto.

C++ · Constructor de clase padre y clase hija
#include <iostream>
using namespace std;

class Persona {
protected:
    string nombre;
public:
    Persona(string n) {
        nombre = n;
    }

    void mostrarNombre() {
        cout << "Nombre: " << nombre << endl;
    }
};

class Alumno : public Persona {
private:
    string matricula;
public:
    Alumno(string n, string m) : Persona(n) {
        matricula = m;
    }

    void mostrarAlumno() {
        mostrarNombre();
        cout << "Matrícula: " << matricula << endl;
    }
};

int main() {
    Alumno alumno1("Ana", "A002");
    alumno1.mostrarAlumno();

    return 0;
}
Punto clave del constructor
Alumno(string n, string m) : Persona(n)
Esta línea crea el constructor de Alumno y llama al constructor de Persona. Así se inicializa primero la parte heredada.
protected:
Permite que la clase hija pueda acceder al miembro heredado, pero evita que se use libremente desde fuera de la clase.

06

Herencia múltiple: implementar varias capacidades

La herencia múltiple permite que una clase derive de dos o más clases base. En la práctica puede servir cuando una clase necesita combinar capacidades diferentes.

C++ · Herencia múltiple desde cero
#include <iostream>
using namespace std;

class Impresora {
public:
    void imprimir() {
        cout << "Imprimiendo documento" << endl;
    }
};

class Scanner {
public:
    void escanear() {
        cout << "Escaneando documento" << endl;
    }
};

class Multifuncional : public Impresora, public Scanner {
};

int main() {
    Multifuncional equipo;
    equipo.imprimir();
    equipo.escanear();

    return 0;
}
Código explicado línea por línea
class Impresora { ... };
Define una primera clase base con la capacidad de imprimir.
class Scanner { ... };
Define una segunda clase base con la capacidad de escanear.
class Multifuncional : public Impresora, public Scanner
Crea una clase derivada que hereda de dos clases. La coma separa cada clase padre.
07

Aplicación con herencia múltiple

Ahora se crea un objeto de tipo Multifuncional. Ese objeto podrá usar los métodos heredados de Impresora y de Scanner.

Salida esperada en consola
Imprimiendo documento
Escaneando documento
Cuidado práctico

En herencia múltiple conviene que cada clase padre tenga responsabilidades claras. Si dos clases padre tienen métodos con el mismo nombre, tendrás que especificar de cuál clase viene el método.

01

¿Qué es la Herencia?

La herencia es una propiedad de la Programación Orientada a Objetos que permite crear una clase nueva a partir de otra que ya existe. La nueva clase aprovecha lo que la clase anterior ya sabe hacer y, además, puede agregar sus propias acciones.

💡 Explicación clara

Imagina que ya tienes una clase llamada Animal. Esa clase ya sabe hacer cosas generales como respirar y comer.

Ahora quieres crear una clase Perro. ¿Tiene sentido volver a escribir otra vez que el perro respira y come? ❌ No. Mejor hacemos que Perro herede de Animal. Así el perro ya trae lo básico y solo agregamos lo propio: por ejemplo, ladrar.

Idea clave

La herencia responde a la frase: “es un tipo de”. Si la frase tiene sentido, probablemente hay herencia. Ejemplo: un Perro es un tipo de Animal; una Moto es un tipo de Vehículo.

classSirve para crear un molde. Ese molde define cómo será un objeto.
publicSignifica “esto se puede usar desde afuera”. Es como dejar una puerta abierta.
: publicEn C++ indica que una clase está heredando de otra.
Animal
+ respirar()
+ comer()
⟶
Perro
+ ladrar()
hereda respirar()
C++ · Herencia básica
class Animal {
public:
    void respirar() {
        cout << "Respirando...";
    }
};

class Perro : public Animal {
public:
    void ladrar() {
        cout << "Guau!";
    }
};
Código explicado línea por línea
class Animal {
class indica que se va a crear una clase. Animal es el nombre de la clase padre. La llave { abre el bloque donde se escribirán sus miembros.
public:
public es un modificador de acceso. Significa que lo que se escriba debajo podrá usarse desde fuera de la clase, por ejemplo desde un objeto.
void respirar() {
void significa que el método no regresa ningún valor. respirar() es el nombre del método. Los paréntesis indican que es una función y la llave abre sus instrucciones.
cout << "Respirando...";
cout muestra información en pantalla. << envía el texto hacia la consola. El mensaje entre comillas es lo que verá el usuario. El punto y coma ; termina la instrucción.
}
Esta llave cierra el método respirar(). Todo lo que estaba dentro pertenece a ese método.
};
La llave } cierra la clase Animal. En C++, al terminar una clase se debe escribir punto y coma ;.
class Perro : public Animal {
Perro es la clase hija. Los dos puntos : indican herencia. public Animal significa que Perro hereda públicamente de Animal, por eso podrá usar respirar().
public:
Hace públicos los elementos propios de la clase Perro, como el método ladrar().
void ladrar() {
Declara un método nuevo de la clase hija. void indica que no devuelve nada y ladrar() será una acción propia del perro.
cout << "Guau!";
Muestra en pantalla el texto Guau!. El punto y coma finaliza la instrucción.
}
Cierra el método ladrar().
};
Cierra la clase Perro. El punto y coma es obligatorio al terminar la definición de una clase.
02

Clase Padre y Clase Hija

La clase padre, también llamada clase base, contiene características generales. La clase hija, también llamada clase derivada, toma esas características y agrega detalles más específicos.

🍎 Ejemplo de la vida real

Piensa en una plantilla general llamada Usuario. Todos los usuarios pueden iniciar sesión.

Pero un Alumno también puede consultar calificaciones, y un Docente puede capturar calificaciones. Entonces Alumno y Docente pueden heredar lo general de Usuario y agregar sus propias funciones.

ConceptoTambién se conoce comoEjemplo
Clase padreClase base / SuperclaseAnimal
Clase hijaClase derivada / SubclasePerro
HerenciaRelación “es un tipo de”Perro : public Animal
En C++

La sintaxis principal es class Hija : public Padre. Léelo así: “voy a crear una clase hija que toma lo público de la clase padre”.

03

Reutilización de Código

Uno de los beneficios más importantes de la herencia es evitar repetir código. En lugar de escribir la misma instrucción en varias clases, se escribe una sola vez en la clase padre y las clases hijas la aprovechan.

🍐 ¿Por qué esto importa?

Si copias y pegas el mismo código en muchas clases, después cualquier cambio se vuelve un dolor de cabeza. Tendrías que corregir lo mismo en todos lados.

Con herencia, pones lo común en una clase padre. Así, si algo cambia, lo corriges en un solo lugar.

C++ · Reutilización de métodos
class Usuario {
public:
    void iniciarSesion() {
        cout << "Sesión iniciada";
    }
};

class Alumno : public Usuario {
public:
    void consultarCalificacion() {
        cout << "Mostrando calificaciones";
    }
};
Código explicado línea por línea
class Usuario {
Crea una clase base llamada Usuario. Aquí se colocan acciones generales que pueden compartir otros tipos de usuario.
public:
Permite que los métodos declarados debajo puedan ejecutarse desde objetos o clases hijas.
void iniciarSesion() {
Crea un método llamado iniciarSesion(). Como es void, solo realiza una acción y no devuelve un dato.
cout << "Sesión iniciada";
Muestra el mensaje en consola. Esta línea representa una acción común que podrá reutilizar la clase hija.
}
Cierra el método iniciarSesion().
};
Cierra la clase Usuario. El ; final es obligatorio en C++.
class Alumno : public Usuario {
Crea la clase hija Alumno. Con : public Usuario hereda lo público de Usuario, por ejemplo iniciarSesion().
public:
Los métodos escritos debajo serán públicos dentro de Alumno.
void consultarCalificacion() {
Agrega un método nuevo exclusivo del alumno. La clase hija no solo hereda, también puede extenderse con funciones propias.
cout << "Mostrando calificaciones";
Imprime un mensaje que simula la consulta de calificaciones.
}
Cierra el método consultarCalificacion().
};
Cierra la clase Alumno.
Ventaja

Si mañana cambia la forma de iniciar sesión, solo se modifica en la clase Usuario. Los alumnos, docentes o administradores que hereden de Usuario aprovecharán ese cambio automáticamente.

04

Extensibilidad

La herencia permite extender un sistema sin empezar desde cero. Una clase hija conserva lo que ya funciona en la clase padre y agrega nuevas acciones propias.

🍎 Dicho fácil

Es como comprar una mochila básica y después agregarle compartimentos especiales. No tiras la mochila; solo la mejoras.

En programación pasa igual: no borras la clase base, solo creas una clase hija más específica.

Ejemplo sencillo

Si ya tienes una clase Vehiculo, puedes crear Auto, Moto y Camioneta sin duplicar los métodos generales como encender() o frenar().

05

Herencia Simple

La herencia simple ocurre cuando una clase hija hereda de una sola clase padre. Es el caso más directo: una clase toma características de otra.

🍐 Para no confundirse

En herencia simple hay una relación clara: Moto es un tipo de Vehículo.

Entonces Moto puede usar lo general de Vehiculo, como encender(), y también tener algo propio, como acelerar().

C++ · Herencia simple
class Vehiculo {
public:
    void encender() {
        cout << "Vehículo encendido";
    }
};

class Moto : public Vehiculo {
public:
    void acelerar() {
        cout << "Acelerando moto";
    }
};
Código explicado línea por línea
class Vehiculo {
Define la clase padre Vehiculo. Contendrá características generales de cualquier vehículo.
public:
Permite que sus métodos públicos sean accesibles desde objetos y desde clases hijas.
void encender() {
Declara el método encender(). Es una acción general que también podrá usar una Moto.
cout << "Vehículo encendido";
Muestra en pantalla que el vehículo fue encendido.
}
Cierra el método encender().
};
Cierra la clase padre Vehiculo.
class Moto : public Vehiculo {
Crea la clase hija Moto. La expresión : public Vehiculo significa que Moto hereda de Vehiculo.
public:
Indica que los métodos propios de Moto serán públicos.
void acelerar() {
Agrega un método propio de la clase Moto. No viene de la clase padre; es una extensión.
cout << "Acelerando moto";
Muestra el mensaje de aceleración en la consola.
}
Cierra el método acelerar().
};
Cierra la clase Moto.
06

Herencia Múltiple

La herencia múltiple ocurre cuando una clase hija hereda de más de una clase padre. C++ sí permite este tipo de herencia, pero debe usarse con cuidado porque puede generar confusión.

🍎 Ejemplo sencillo

Una impresora multifuncional puede hacer dos cosas: imprimir y escanear.

En código, podríamos decir que Multifuncional hereda de Impresora y también de Scanner. Así obtiene capacidades de ambas clases.

C++ · Herencia múltiple
class Impresora {
public:
    void imprimir() { cout << "Imprimiendo"; }
};

class Scanner {
public:
    void escanear() { cout << "Escaneando"; }
};

class Multifuncional : public Impresora, public Scanner { };
Código explicado línea por línea
class Impresora {
Crea una clase padre llamada Impresora. Representa una capacidad: imprimir.
public:
Hace público el método que se declarará en esta clase.
void imprimir() { cout << "Imprimiendo"; }
Declara el método imprimir() en una sola línea. Al ejecutarse, muestra el texto Imprimiendo.
};
Cierra la clase Impresora.
class Scanner {
Crea otra clase padre llamada Scanner. Representa otra capacidad: escanear.
public:
Permite que su método pueda usarse públicamente.
void escanear() { cout << "Escaneando"; }
Declara el método escanear(), que imprime el mensaje Escaneando.
};
Cierra la clase Scanner.
class Multifuncional : public Impresora, public Scanner { };
Crea una clase hija que hereda de dos clases padre. La coma separa las clases heredadas. Ahora Multifuncional puede imprimir y escanear.
Cuidado

La herencia múltiple puede ser útil, pero úsala con planeación. Si varias clases padre tienen métodos parecidos o con el mismo nombre, el programa puede confundirse y necesitar que le digas exactamente cuál usar.

07

Ambigüedad en Herencia

La ambigüedad aparece cuando una clase hija recibe métodos con el mismo nombre desde diferentes clases padre. En ese caso, C++ necesita que el programador aclare de qué clase quiere usar el método.

🍐 Imagínalo así

Supón que dos maestros te dicen “haz la tarea”, pero cada uno se refiere a una tarea diferente. Tú tendrías que preguntar: “¿la tarea de cuál materia?”.

En C++ pasa algo parecido. Si A y B tienen un método llamado mostrar(), y C hereda de ambas, entonces debes aclarar si quieres usar A::mostrar() o B::mostrar().

C++ · Resolución de ambigüedad
class A { public: void mostrar() { cout << "A"; } };
class B { public: void mostrar() { cout << "B"; } };

class C : public A, public B { };

C obj;
obj.A::mostrar(); // Se especifica que se usará el método de A
Código explicado con calma
class A { ... mostrar() ... };
La clase A tiene un método llamado mostrar(). Ese método imprime la letra A.
class B { ... mostrar() ... };
La clase B también tiene un método llamado mostrar(). Aquí empieza el posible conflicto porque ambas clases usan el mismo nombre.
class C : public A, public B { };
La clase C hereda de A y de B. Por eso recibe dos métodos con el mismo nombre: A::mostrar() y B::mostrar().
C obj;
Crea un objeto llamado obj de la clase C.
obj.A::mostrar();
Aquí se usa el operador :: para decirle a C++: “usa el método mostrar() que viene de la clase A, no el de B”.

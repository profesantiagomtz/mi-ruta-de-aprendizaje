D-01

¿Qué es una línea de base de red?

Una línea de base (baseline) es una fotografía del estado de la red cuando todo funciona correctamente. Es un punto de referencia: si en el futuro algo falla, se compara el estado actual con la línea de base para identificar qué cambió.

Explicación clara

Imagina que tu cuerpo en condiciones normales tiene 36.5 °C de temperatura. Cuando te enfermas, el médico mide 38.5 °C y ya sabe que algo está mal porque tiene una referencia. En redes, la línea de base funciona igual: guardas el estado "sano" de tu red para poder comparar después cuando algo falle.

¿Sin línea de base?¿Con línea de base?
Difícil saber si el rendimiento actual es normal o anormal.Puedes comparar y detectar cambios con datos reales.
La resolución de problemas toma más tiempo.El diagnóstico es más rápido y preciso.
No sabes qué configuraciones estaban activas antes del fallo.Tienes registros confiables de la configuración previa.
Dato importante

Una línea de base documenta dos cosas principales: el estado del host (la computadora del usuario) y el estado del IOS (el router o switch). Ambas capturas juntas forman el registro completo de la red.

D-02

Captura de host

La captura de host consiste en documentar la configuración de red de una computadora: su dirección IP, máscara de subred, gateway predeterminado, servidores DNS y el estado de las interfaces. Esta información se obtiene con comandos ejecutados desde la PC.

¿Por qué documentar el host?

Si un alumno cambia accidentalmente la IP de su PC o el gateway en clase, sin una línea de base no sabes cómo era la configuración original. Con la captura guardada, restauras los valores correctos en segundos.

D-03

Comandos clave para la captura del host

Además de ipconfig /all, existen otros comandos de Windows que complementan la documentación del estado del host y son esenciales para una captura completa.

Comandos de captura en Windows
ipconfig /all
Muestra toda la configuración IP: dirección, máscara, gateway, DNS, MAC, DHCP activo o no. Es el comando principal para la captura de host.
ping 127.0.0.1
Verifica que la pila TCP/IP del equipo esté funcionando correctamente (prueba de loopback). Si falla, el problema es local.
ping 192.168.1.1
Verifica conectividad con el gateway (router). Si falla, el equipo no puede salir de la red local.
arp -a
Muestra la tabla ARP: qué IPs conocidas tiene el equipo y a qué MACs corresponden. Útil para detectar conflictos de IP.
netstat -n
Muestra las conexiones activas del equipo. Sirve para documentar qué servicios o sesiones estaban abiertas en el momento de la captura.
Tabla ARP del host
C:\> arp -a

Interface: 192.168.1.10 --- 0x4
  Internet Address      Physical Address      Type
  192.168.1.1           00-aa-bb-cc-dd-ee     dynamic
  192.168.1.20          00-11-22-33-44-55     dynamic
  192.168.1.255         ff-ff-ff-ff-ff-ff     static
Consejo de laboratorio

Cuando documentes una línea de base en Packet Tracer, captura pantalla de cada comando ejecutado. Guarda las imágenes con nombres descriptivos como host_ipconfig_all.png, host_arp.png, etc. Eso facilita comparar después si algo cambia.

D-04

Captura de IOS

La captura de IOS documenta el estado del router o switch: su configuración activa, las interfaces, las rutas y el estado general del dispositivo. Se realiza desde la consola del dispositivo usando comandos de IOS.

¿Por qué documentar el IOS?

Si alguien ejecuta el comando erase startup-config o el router sufre un reinicio inesperado, perdemos toda la configuración. Con la captura de IOS guardada (en un archivo de texto o en un servidor TFTP), podemos restaurarla fácilmente.

Router – Configuración activa
Router# show running-config

Building configuration...

Current configuration : 842 bytes
!

hostname R1-LAB
!
enable secret 5 $1$mERr$hx5rVt7rPNoS4wqbXKX7m0
!
interface GigabitEthernet0/0

!

 ip address 10.0.0.1 255.255.255.252

!
ip route 0.0.0.0 0.0.0.0 10.0.0.2

end
Running-config vs Startup-config

running-config es la configuración que está en uso ahora mismo (en RAM). startup-config es la configuración guardada (en NVRAM) que se carga al encender el router. Para que ambas sean iguales hay que ejecutar copy running-config startup-config.

D-05

Comandos clave para la captura de IOS

En IOS Cisco existen varios comandos show que permiten documentar el estado completo del dispositivo. Juntos forman la captura de IOS de una línea de base profesional.

Comandos show para la captura de IOS
show running-config
Muestra toda la configuración activa del dispositivo. Es el comando más importante para documentar el estado del router o switch.
show startup-config
Muestra la configuración guardada en NVRAM. Si es diferente a la running-config, significa que hay cambios sin guardar.
show ip interface brief
Muestra un resumen de todas las interfaces: dirección IP, estado (up/down) y protocolo. Esencial para la captura de IOS.
show version
Muestra la versión del IOS, el tiempo activo del dispositivo (uptime), la memoria RAM/Flash, y el modelo del hardware.
show ip route
Muestra la tabla de enrutamiento: qué redes conoce el router y cómo llegar a ellas. Parte fundamental de la línea de base de un router.
show interfaces
Información detallada de cada interfaz: velocidad, encapsulación, errores, paquetes enviados y recibidos. Sirve para detectar problemas físicos o de rendimiento.

Dato del show version¿Qué informa?
Versión IOSLa versión exacta del sistema operativo del dispositivo.
UptimeCuánto tiempo lleva encendido sin reiniciarse. Útil para saber si hubo reinicios recientes.
RAM / FlashCuánta memoria tiene disponible para almacenar IOS y configuraciones.
ModeloEl hardware exacto del equipo. Necesario para buscar compatibilidad de IOS.
Proceso recomendado para la línea de base

Ejecuta los comandos en este orden: show version → show running-config → show ip interface brief → show ip route. Guarda la salida completa en un archivo de texto con fecha. Eso es tu línea de base de IOS.

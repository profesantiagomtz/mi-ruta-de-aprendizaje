A-01

¿Qué es un Router ISR?

Un Router ISR (Integrated Services Router — Router de Servicios Integrados) es un dispositivo de red fabricado por Cisco que combina en un solo equipo múltiples funciones: enrutamiento, switching, seguridad, telefonía y acceso WAN. Es el tipo de router más usado en empresas pequeñas y medianas.

Explicación clara

Imagina que antes necesitabas un router para conectarte a internet, un switch para conectar las PCs, un firewall para seguridad y otro equipo para el teléfono. El ISR mete todo eso en una sola caja. Es como comprar una impresora multifuncional: imprime, escanea y fotocopia desde un solo dispositivo.

Función¿Qué hace en el ISR?
EnrutamientoDirige el tráfico entre redes (LAN, WAN, Internet).
SwitchingAlgunos modelos incluyen puertos de switch integrados para la LAN.
SeguridadFirewall, VPN y control de acceso integrados.
Acceso WANPuertos seriales, DSL o cable para conectarse al proveedor de internet.
Telefonía IPAlgunos modelos soportan llamadas por internet (VoIP).
Modelos comunes en el laboratorio

En Cisco Packet Tracer trabajarás principalmente con el Router 1941 y el Router 2911. Ambos son modelos ISR de segunda generación (ISR G2) con soporte para múltiples módulos de expansión.

A-02

Configuración física del ISR

Antes de configurar un ISR por software, es importante conocer su parte física: los puertos disponibles, cómo conectar los cables y qué módulos se pueden agregar para ampliar sus capacidades.

Componentes físicos principales

Al frente del router ISR encontrarás puertos GigabitEthernet (para la LAN), puertos seriales (para conexiones WAN con el proveedor), un puerto de consola (RJ-45 azul, para configuración directa), un puerto auxiliar (AUX, para acceso remoto por módem) y ranuras para módulos de expansión (HWIC).

Puerto / ComponenteTipo de cable¿Para qué se usa?
GigabitEthernet (G0/0, G0/1)Cable directo o cruzadoConexión a la LAN (switch, PCs).
Serial (S0/0/0, S0/0/1)Cable serial DCE/DTEConexión WAN al proveedor de internet.
Consola (CON)Cable de consola (rollover)Configuración inicial sin red.
Auxiliar (AUX)Cable telefónico RJ-11Acceso remoto de emergencia por módem.
USBCable USB tipo ACargar imágenes IOS o configuraciones.
Cable DCE vs DTE

En las conexiones seriales WAN, uno de los dos routers debe ser el DCE (proporciona la señal de reloj) y el otro el DTE. En el laboratorio con Packet Tracer, el extremo DCE se identifica con un pequeño ícono de reloj. El router DCE debe configurar la velocidad del enlace con el comando clock rate 64000.

A-03

Acceso dentro de banda y fuera de banda

Existen dos formas de acceder a la configuración de un router ISR. La elección depende de si la red está funcionando o si el dispositivo está fuera de línea.

Analogía

Imagina que el router es un servidor en un cuarto de telecomunicaciones. Si la red funciona, lo administras desde tu PC por la red (dentro de banda). Si la red está caída o el router nunca fue configurado, necesitas ir físicamente al cuarto y conectarte con un cable de consola (fuera de banda).

Tipo de acceso¿Cómo se conecta?¿Cuándo se usa?Requiere red
Dentro de bandaTelnet, SSH, SDM (por la red)Administración rutinaria cuando la red funciona.Sí ✅
Fuera de bandaCable de consola (CON) o puerto AUXConfiguración inicial o cuando la red está caída.No ❌
Recomendación de seguridad

Para acceso dentro de banda siempre usa SSH en lugar de Telnet. SSH cifra toda la comunicación, mientras que Telnet envía usuario y contraseña en texto claro — cualquiera que esté en la red puede interceptarlos con un analizador de paquetes.

Acceso por SSH — Configuración básica en IOS
Router(config)# ip domain-name conalep.local          ← nombre de dominio requerido
Router(config)# crypto key generate rsa                   ← genera las llaves de cifrado
How many bits in the modulus [512]: 1024              ← 1024 bits mínimo recomendado
Router(config)# username admin secret Admin2025      ← usuario local
Router(config)# line vty 0 4
Router(config-line)# transport input ssh               ← solo permite SSH
Router(config-line)# login local                       ← usa usuario local
A-04

CLI vs SDM — Dos formas de configurar el router

El ISR puede configurarse de dos maneras: mediante la CLI (interfaz de línea de comandos) que ya conoces del R.A. 2.1, o mediante el SDM (Security Device Manager), una interfaz gráfica web que simplifica la configuración para usuarios menos experimentados.

CaracterísticaCLISDM
Tipo de interfazTexto — comandos escritosGráfica — menús y formularios web
AccesoConsola, Telnet o SSHNavegador web (dentro de banda)
VelocidadRápido para expertosMás intuitivo para principiantes
PrecisiónControl total sobre cada parámetroLimitado a las opciones del asistente
Uso en producciónEl estándar en la industriaConfiguración inicial o básica
¿Cuál debo aprender?

En el mundo laboral real, el 95% de los técnicos de red usa la CLI. El SDM es útil para configuraciones rápidas o cuando un cliente no técnico necesita administrar su propio router. Dominar la CLI siempre te dará más ventaja profesional.

SDM en Packet Tracer

Cisco Packet Tracer no incluye el SDM real, pero simula su funcionalidad en la pestaña Config de cada dispositivo. Esta pestaña muestra los mismos parámetros que configuraría el SDM: interfaces, rutas, DHCP, NAT, etc., en un formato gráfico amigable.

A-05

Archivos de configuración del ISR

El ISR maneja dos archivos de configuración importantes. Entender la diferencia entre ellos es fundamental para no perder configuraciones al reiniciar el equipo.

Los dos archivos de configuración
show running-config
Configuración en ejecución — vive en la RAM. Es la configuración activa en este momento. Se pierde si el router se apaga sin guardar.
show startup-config
Configuración de inicio — vive en la NVRAM. Es lo que se carga cuando el router enciende. No cambia hasta que se guarda explícitamente.
copy running-config startup-config
Guarda la configuración activa (RAM) en la NVRAM. Después de cada cambio importante siempre ejecuta este comando.
copy startup-config running-config
Carga la configuración guardada sobre la activa. Útil para deshacer cambios recientes no guardados.
erase startup-config
Borra la configuración de inicio. El router arrancará sin configuración en el siguiente reinicio. Usar con mucho cuidado.
reload
Reinicia el router. Si hay cambios sin guardar en la running-config, IOS pregunta si deseas guardarlos antes de reiniciar.
Error común

Muchos estudiantes configuran el router correctamente pero olvidan ejecutar copy running-config startup-config. Al apagar Packet Tracer y reabrir el archivo, el router arranca sin su configuración. Siempre guarda antes de cerrar.

Verificar si la configuración está guardada
Router# show running-config ← configuración ACTIVA en RAM

hostname ISR-LAB
enable secret 5 $1$mERr$...
interface GigabitEthernet0/0
 ip address 192.168.1.1 255.255.255.0
 no shutdown

Router# show startup-config ← configuración GUARDADA en NVRAM

startup-config is not present   ← ¡Alerta! No se ha guardado nada

Router# copy running-config startup-config
Destination filename [startup-config]?
Building configuration...
[OK]   ← Ahora sí está guardada
A-06

SDM Express, conexión WAN y NAT dinámica

El SDM Express es el asistente de configuración rápida del SDM. Guía al administrador paso a paso para configurar los aspectos básicos del router: interfaces, acceso WAN e internet, y servicios como NAT y DHCP.

¿Qué es NAT?

NAT (Network Address Translation) es la tecnología que permite que muchas computadoras de una red local compartan una sola dirección IP pública para salir a internet. Es como un edificio de departamentos donde todos tienen su propio número interno (IP privada) pero al exterior solo aparece una dirección (IP pública). Sin NAT, cada PC necesitaría su propia IP pública — algo imposible hoy en día por la escasez de direcciones IPv4.

Tipo de NAT¿Cómo funciona?¿Cuándo se usa?
NAT estáticoUna IP privada específica siempre se traduce a la misma IP pública.Servidores internos que deben ser accesibles desde internet.
NAT dinámicoLas IPs privadas se traducen a un conjunto de IPs públicas disponibles.Empresas con varias IPs públicas y muchos usuarios.
PAT / NAT con sobrecargaMuchas IPs privadas comparten UNA sola IP pública usando diferentes puertos.Hogares y pequeñas empresas. Es el tipo más común.
Configuración de NAT dinámico con sobrecarga (PAT) en CLI
── Paso 1: Definir qué IPs serán traducidas (red interna)
Router(config)# ip nat pool POOL-WAN 200.1.1.1 200.1.1.1 netmask 255.255.255.0
Router(config)# access-list 1 permit 192.168.1.0 0.0.0.255

── Paso 2: Activar NAT con sobrecarga
Router(config)# ip nat inside source list 1 pool POOL-WAN overload

── Paso 3: Marcar interfaz interna e interfaz externa
Router(config)# interface GigabitEthernet0/0
Router(config-if)# ip nat inside      ← LAN (red interna)
Router(config)# interface Serial0/0/0
Router(config-if)# ip nat outside     ← WAN (salida a internet)
Verificación de NAT

Usa show ip nat translations para ver en tiempo real qué IPs internas están siendo traducidas y a qué puerto externo corresponden. Si ves entradas en la tabla, el NAT está funcionando correctamente.

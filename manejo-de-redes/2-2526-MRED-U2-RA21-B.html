B-01

Aplicación de nombres a dispositivos

Asignar un nombre ayuda a identificar cada router o switch. Si en una práctica tienes varios equipos y todos se llaman Router, será fácil confundirse.

Cable de consola

Útil para la configuración inicial cuando el dispositivo todavía no tiene acceso remoto habilitado.

Switch de red

Conecta computadoras, impresoras, access points y otros dispositivos dentro de la LAN.

Router

Interconecta redes y permite configurar interfaces, rutas, contraseñas y servicios.

IOS – Asignar hostname
Router> enable
Router# configure terminal
Router(config)# hostname R1-LAB
R1-LAB(config)#
Explicación línea por línea
Router> enable
Entramos al modo privilegiado.
Router# configure terminal
Entramos al modo de configuración global.
hostname R1-LAB
Cambia el nombre del dispositivo a R1-LAB.
R1-LAB(config)#
El prompt cambia de inmediato, confirmando que el nombre ya fue aplicado.
B-02

Limitación del acceso y contraseñas

Configurar contraseñas es proteger la entrada al dispositivo. Hay contraseña para consola, para modo privilegiado y para acceso remoto.

IOS – Consola, enable secret y VTY
R1-LAB(config)# line console 0
R1-LAB(config-line)# password Cisco123
R1-LAB(config-line)# login
R1-LAB(config-line)# exit

R1-LAB(config)# enable secret Seguro123

R1-LAB(config)# line vty 0 4
R1-LAB(config-line)# password Remoto123
R1-LAB(config-line)# login
Diferencia clave

password establece la clave, pero login es el comando que obliga al dispositivo a pedirla. Si olvidas login, la contraseña queda configurada pero no se solicita.

B-03

Encriptación de contraseñas y mensajes de aviso

Para evitar que las contraseñas aparezcan en texto claro en la configuración, se usa service password-encryption. También se recomienda configurar un banner de acceso restringido.

IOS – Encriptación y banner
R1-LAB(config)# service password-encryption
R1-LAB(config)# banner motd #
ACCESO RESTRINGIDO. SOLO PERSONAL AUTORIZADO.
#
Cuidado

No uses banners como “Bienvenido”. Es mejor usar mensajes claros de acceso restringido.

B-04

Administración de archivos de configuración

El router trabaja con una configuración activa y una configuración guardada. Si configuras algo y no lo guardas, puedes perderlo al reiniciar.

ArchivoUbicaciónDescripción
running-configRAMConfiguración activa. Se pierde si no se guarda.
startup-configNVRAMConfiguración guardada que carga al iniciar.
IOS – Guardar, respaldar y restaurar
R1-LAB# copy running-config startup-config   ! Guardar cambios
R1-LAB# show running-config                  ! Ver config activa
R1-LAB# show startup-config                  ! Ver config guardada
R1-LAB# erase startup-config                 ! Borrar config guardada
R1-LAB# copy running-config tftp:            ! Respaldar en servidor TFTP
B-05

Configuración de interfaces

Una interfaz es el punto por donde el equipo se conecta a la red. En routers, muchas interfaces vienen apagadas por seguridad y deben activarse con no shutdown.

Puertos Ethernet activos

Las luces de enlace ayudan a verificar que existe conexión física entre dispositivos.

Consola para configuración

Antes de configurar una IP o SSH, normalmente se accede por consola.

IOS – Interfaz Ethernet del router y VLAN del switch
R1-LAB(config)# interface gigabitEthernet 0/0
R1-LAB(config-if)# ip address 192.168.1.1 255.255.255.0
R1-LAB(config-if)# no shutdown

SW1(config)# interface vlan 1
SW1(config-if)# ip address 192.168.1.254 255.255.255.0
SW1(config-if)# no shutdown
SW1(config)# ip default-gateway 192.168.1.1
Explicación rápida
interface gigabitEthernet 0/0
Selecciona el puerto físico del router que se va a configurar.
ip address 192.168.1.1 255.255.255.0
Asigna dirección IP y máscara a la interfaz.
no shutdown
Enciende la interfaz. Sin este comando puede quedar apagada.
interface vlan 1
Selecciona la interfaz virtual de administración del switch.
ip default-gateway 192.168.1.1
Le indica al switch por dónde salir cuando necesite comunicarse fuera de su red.

F1

Modos de acceso y configuración básica

Contexto

Llegaste al laboratorio. El router está encendido. Solo hay un cable de consola disponible conectado a PC-LAB-01. El hostname sigue siendo Router y no hay contraseñas configuradas.

Fase 18 preguntas
¿Por qué en este momento es IMPOSIBLE usar SSH o Telnet para entrar al router?
El router está recién instalado, sin configuración previa.
El cable de consola bloquea SSH
Las interfaces no tienen IP y el acceso VTY no está configurado
Packet Tracer no soporta SSH ni Telnet
Al abrir la terminal del router por consola, el primer prompt que ves es Router>. ¿Qué modo representa y qué puedes hacer ahí?
Modo privilegiado — puedes cambiar toda la configuración
Modo EXEC de usuario — solo comandos básicos de visualización
Modo configuración global — puedes cambiar hostname y contraseñas
¿Qué comando escribes para subir de modo usuario a modo privilegiado?
El director pide que el router se llame CONALEP-NP. ¿En qué modo IOS se configura el hostname?
Router> (modo usuario)
Router# (modo privilegiado)
Router(config)# (configuración global)
Para proteger el acceso al modo privilegiado, escribes enable secret Admin2025. ¿Por qué se prefiere enable secret sobre enable password?
enable secret solo funciona con cable de consola
enable secret cifra la contraseña con hash; enable password puede quedar en texto claro
enable password ya no funciona en versiones modernas de IOS
Escribe el comando para configurar un mensaje de aviso (banner) en el router. El mensaje debe ser: ACCESO RESTRINGIDO
Pista: el comando usa un carácter delimitador antes y después del mensaje.
Terminaste de configurar el router. ¿Qué comando guardas para que la configuración sobreviva un reinicio?
copy startup-config running-config
copy running-config startup-config
erase startup-config
El técnico anterior configuró todo pero NO guardó. Apagaron el router para moverlo. ¿Qué pasó con la configuración al volver a encenderlo?
El router cargó la configuración desde la RAM automáticamente
La configuración se perdió porque la RAM es volátil y no se guardó en NVRAM
IOS guarda automáticamente cada 5 minutos en NVRAM

F2

Diagnóstico de conectividad

Contexto

Ya entraste al router. Ahora el director te dice: "las PCs del laboratorio no tienen internet." Ejecutas show ip interface brief y ves esto:

Router — show ip interface brief
CONALEP-NP# show ip interface brief

Interface              IP-Address      OK? Method Status                Protocol
GigabitEthernet0/0     10.0.0.1        YES manual up                    up
GigabitEthernet0/1     unassigned      YES unset  administratively down down
Fase 26 preguntas
Analiza el output anterior. ¿Cuál es exactamente el problema con la interfaz G0/1?
El cable de red está desconectado físicamente
La interfaz está dañada y debe reemplazarse
La interfaz está apagada por configuración y sin IP asignada
¿Qué comando activa una interfaz que está en estado administratively down?
La PC-LAB-01 tiene IP 192.168.10.5/24 y gateway 192.168.10.1. Un alumno hace ping 127.0.0.1 y responde bien, pero ping 192.168.10.1 falla. ¿Qué significa eso?
La PC funciona bien. El problema está en el router (G0/1 no responde)
La tarjeta de red de la PC está dañada
El ping a 127.0.0.1 prueba la conexión a internet
Después de corregir G0/1, el ping al gateway ya responde. Pero la PC aún no puede llegar a 10.0.0.10 (un servidor externo). En el proceso de verificación de 6 pasos, ¿en qué paso estás fallando?
Paso 2 — prueba de loopback local
Paso 4 — prueba al gateway
Paso 6 — prueba a host remoto
Un ping desde el router muestra ..... (cinco puntos). ¿Qué indica cada punto en la CLI de Cisco IOS?
Cada punto indica un paquete exitoso
Cada punto representa un timeout — el paquete no recibió respuesta
Cada punto indica que el destino está apagado (unreachable)
¿Cuál es el estado correcto que debe mostrar una interfaz completamente operativa en el output de show ip interface brief?
up / down
administratively down / down
up / up

¿Qué es una línea de base de red?
Un diagrama de cómo están conectados los dispositivos
Un registro del estado normal de la red para usar como punto de comparación cuando algo falle
Una lista de contraseñas y usuarios del router
¿Qué comando ejecutas en Windows para capturar la información completa de red de una PC (IP, máscara, gateway, DNS, MAC)?
Analiza este output de ipconfig /all. ¿Qué está mal?
IPv4 Address: 192.168.10.5
Subnet Mask: 255.255.0.0
Default Gateway: (en blanco)
DHCP Enabled: Yes
La dirección IP no es válida para esta red
La máscara es incorrecta (/16 en lugar de /24) y falta el gateway
Solo falta el servidor DNS
¿Cuál es el comando del router que muestra la configuración activa completa — ideal para incluir en la línea de base?
¿Por qué hubiera sido tan útil tener una línea de base antes de que el técnico anterior se fuera sin documentar?
Con la línea de base hubiéramos tenido las contraseñas
Tendríamos las IPs, el estado de las interfaces y la configuración de las PCs ya documentados — no habría necesidad de reconstruir todo desde cero
La línea de base habría evitado que el técnico cometiera errores
Para completar la línea de base del router, ¿cuál de estos comandos te da el resumen más rápido del estado actual de todas las interfaces (IP, estado y protocolo)?

📋 Instrucciones de la práctica — léelas completas antes de abrir Packet Tracer
PASO 1 Abre el archivo en Packet Tracer

Descarga el archivo de arriba. Luego abre Cisco Packet Tracer y haz doble clic sobre el archivo descargado — se abrirá automáticamente.

Verás una red con: 2 PCs → 1 Switch → 1 Router.
La red tiene errores intencionales que tú debes encontrar y corregir.
PASO 2 Corrige el gateway de las PCs

Las dos PCs no tienen gateway configurado. Debes hacerlo en PC-LAB-01 y PC-LAB-02.

① Haz clic en PC-LAB-01
② Ve a la pestaña Desktop
③ Haz clic en IP Configuration
④ En el campo Default Gateway escribe: 192.168.10.1
⑤ Cierra la ventana y repite lo mismo en PC-LAB-02
PASO 3 Entra al router por consola

El router se llama Router (sin nombre). Tienes que entrar a su CLI y configurarlo.

① Haz clic en el router (el dispositivo grande de la derecha)
② Ve a la pestaña CLI
③ Si aparece una pregunta en inglés, escribe no y presiona Enter
④ Presiona Enter una vez más hasta ver: Router>
PASO 4 Configura el router — escribe estos comandos en orden

Escribe un comando a la vez y presiona Enter después de cada uno. El prompt cambia conforme avanzas — eso es normal.

Router> enable ← sube a modo privilegiado
Router# configure terminal ← entra a configuración
Router(config)# hostname CONALEP-NP ← cambia el nombre
CONALEP-NP(config)# enable secret Admin2025 ← contraseña segura
CONALEP-NP(config)# banner motd # ACCESO RESTRINGIDO. SOLO PERSONAL AUTORIZADO. #
CONALEP-NP(config)# interface GigabitEthernet0/1 ← selecciona la interfaz
CONALEP-NP(config-if)# ip address 192.168.10.1 255.255.255.0 ← asigna IP
CONALEP-NP(config-if)# no shutdown ← enciende la interfaz
CONALEP-NP(config-if)# end ← regresa a modo privilegiado
CONALEP-NP# copy running-config startup-config ← guarda todo
Cuando pregunte Destination filename [startup-config]? solo presiona Enter
PASO 5 Verifica que todo funciona

Desde el router, escribe este comando y confirma que G0/1 ya aparece up / up:

CONALEP-NP# show ip interface brief
Debes ver G0/1 con IP 192.168.10.1 y estado up / up

Luego ve a PC-LAB-01 → Desktop → Command Prompt y prueba:

C:\> ping 192.168.10.1 ← debe responder con Reply from...
C:\> ping 192.168.10.6 ← debe responder (PC-LAB-02)
PASO 6 Documenta la línea de base

Antes de entregar, captura el estado final de la red. En el router escribe:

CONALEP-NP# show running-config
CONALEP-NP# show ip interface brief

En PC-LAB-01 → Desktop → Command Prompt:

C:\> ipconfig /all
